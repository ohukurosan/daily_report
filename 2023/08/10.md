## わかったこと
- **touchコマンド**
    
    touchコマンドは、新しい空のファイルを作成するために使用されます。また、既存のファイルの最終更新日を変更することもできます。
    
    - 使用例
        
        例1：`$ touch file.txt`
        
        このコマンドは、`file.txt` という名前の新しい空のファイルを作成します。
        
        例2：`$ touch -t 202201012359.59 file.txt`
        
        このコマンドは、`file.txt` ファイルの最終更新日を2022年1月1日23時59分に設定します。
        
    - touchコマンドのオプション
        - **a**
            
            このオプションは、アクセス時刻を変更します。
            
            例: `$ touch -a file.txt`
            
            このコマンドは、`file.txt` ファイルの最終アクセス日時を現在の日時に更新します。
            
        - **m**
            
            このオプションは、変更時刻を変更します。
            
            例: `$ touch -m file.txt`
            
            このコマンドは、`file.txt` ファイルの最終変更日時を現在の日時に更新します。
            
        - **t**
            
            このオプションは、指定した時刻を使用して最終更新日時を設定します。
            
            例: `$ touch -t 202201012359.59 file.txt`
            
            このコマンドは、`file.txt` ファイルの最終更新日時を2022年1月1日23時59分に設定します。
            
- **ヘルプの使い方**
    
    `-help`オプションは、Linuxコマンドで一般的なオプションであり、コマンドの使用方法や利用可能なオプションの概要を表示します。通常、以下の形式で使用されます。
    
    `$ command --help`
    
- **特別なディレクトリ**
    
    Linux では、ディレクトリの中で特別なディレクトリや、それを表す記号があり、以下がそ
    の代表的なもの
    
    - `.`: 現在のディレクトリを指します。つまり、現在作業しているディレクトリを指します。
    - `..`: 親ディレクトリを指します。つまり、現在のディレクトリを含むディレクトリを指します。
    - `~`: 現在のユーザーのホームディレクトリを指します。
    - `/`: ファイルシステムのルートディレクトリを指します。
    
- **絶対 (パス) 指定と相対 (パス) 指定**
    - **絶対パス**
        
        絶対パスは、ルートディレクトリから特定のファイルまたはディレクトリまでの完全なパスです。常にスラッシュ（/）で始まります。例えば、`/home/user/Documents/file.txt`は、システム上の`user`アカウントの`Documents`ディレクトリ内の`file.txt`を指定する絶対パスです。
        
    - **相対パス**
        
        相対パスは、現在の作業ディレクトリに対する相対的なパスです。スラッシュ（/）で始まりません。代わりに、現在のディレクトリからのファイルまたはディレクトリの場所を指定します。例えば、現在の作業ディレクトリが`/home/user/`の場合、相対パス`Documents/file.txt`は、カレントディレクトリにある`user`アカウントの`Documents`ディレクトリ内の`file.txt`を指定します。
        
- **正規表現とパイプ**
    - **標準入出力**
        - **標準入力**
            
            プログラムに入力されるデータのことで、通常はキーボードからの入力が対応します。
            
        - **標準出力**
            
            成功した処理の結果を表示する
            
        - **標準エラー出力**
            
            プログラムからのエラーメッセージを出力する場所であり、一般的にはプログラムを実行した端末のディスプレイに出力される
            
            ※標準出力と標準エラー出力は別々の出力先に出力される
            
    - **リダイレクト**
        - **標準入出力のリダイレクト**
            
            Linux では、標準入力や標準出力をファイルにリダイレクトすることができます。これは、リダイレクト演算子`<`、`>`、`>>`を使用して行います。
            
            - `<`: ファイルから標準入力を読み込むために使用されます。
                
                例：`$ program < input.txt`
                
                このコマンドは、`input.txt` ファイルから標準入力を読み込み、それを `program` コマンドに渡します。
                
            - `>`: 標準出力をファイルに書き込むために使用されます。ファイルが存在しない場合は、新しいファイルが作成されます。ファイルが既に存在する場合は、ファイルの内容が上書きされます。
                
                例：`$ program > output.txt`
                
                このコマンドは、`program` コマンドの標準出力を `output.txt` ファイルに書き込みます。
                
            - `>>`: 標準出力をファイルに追記するために使用されます。ファイルが存在しない場合は、新しいファイルが作成されます。
                
                例：`$ program >> output.txt`
                
                このコマンドは、`program` コマンドの標準出力を `output.txt` ファイルに追記します。
                
            - 標準エラー出力をファイルにリダイレクトするには、`2>` 演算子を使用します。これは `>` 演算子と似ていますが、標準出力ではなく標準エラー出力をリダイレクトします。
                
                例：`$ program 2> error.txt`
                
                このコマンドは、`program` コマンドの標準エラー出力を `error.txt` ファイルにリダイレクトします。
                
        - **パイプ**
            
            パイプは、一つのコマンドの出力を別のコマンドに渡すために使用されます。パイプ演算子`|`を使用して行います。
            
            例：`$ command1 | command2`
            
            このコマンドは、`command1` の出力を `command2` に渡します。`command1` が出力した結果が `command2` の入力として使用されます。
            
            例えば、`ls` コマンドで表示されるファイル一覧を `grep` コマンドで検索する場合、以下のようにパイプを使用して行います。
            
            例：`$ ls | grep keyword`
            
            このコマンドは、`ls` コマンドで表示されるファイル一覧から、`keyword` に一致するファイルを検索します。
            
    - **grepコマンド**
        
        grepコマンドは、指定されたパターンに一致する行を検索するために使用されます。grepは、ファイルの内容または標準入力からパターンを検索します。
        
        - 使用例
            
            例：`$ grep pattern file.txt`
            
            このコマンドは、`file.txt` ファイル内で `pattern` というパターンに一致する行を検索して表示します。
            
        - グローバルオプション
            - **i**
                
                このオプションは、大文字と小文字を区別しないで検索するために使用されます。
                
                例: `$ grep -i pattern file.txt`
                
                このコマンドは、`file.txt` ファイル内で `pattern` というパターンに一致する行を、大文字小文字を区別せずに検索して表示します。
                
            - **v**
                
                このオプションは、指定されたパターンに一致しない行のみを表示するために使用されます。
                
                例: `$ grep -v pattern file.txt`
                
                このコマンドは、`file.txt` ファイル内で `pattern` というパターンに一致しない行を表示します。
                
            - **w**
                
                このオプションは、完全な単語に一致する行のみを表示するために使用されます。
                
                例: `$ grep -w pattern file.txt`
                
                このコマンドは、`file.txt` ファイル内で `pattern` という完全な単語に一致する行を表示します。
                
            - **n**
                
                このオプションは、各行の行番号を表示するために使用されます。
                
                例: `$ grep -n pattern file.txt`
                
                このコマンドは、`file.txt` ファイル内で `pattern` というパターンに一致する行の前に、各行の行番号を表示します。
                
            - **r**
                
                このオプションは、サブディレクトリ内のファイルも再帰的に検索するために使用されます。
                
                例: `$ grep -r pattern directory`
                
                このコマンドは、`directory` ディレクトリ内とそのサブディレクトリ内のファイルで `pattern` というパターンに一致する行を検索して表示します。
                
            - **l**
                
                このオプションは、一致する行が含まれているファイル名のみを表示するために使用されます。
                
                例: `$ grep -l pattern file.txt`
                
                このコマンドは、`file.txt` ファイル内で `pattern` というパターンに一致する行が含まれているファイル名のみを表示します。
                
            - **c**
    - **正規表現**
        
        正規表現は、単純な文字だけでなく、特殊文字を使用してパターンの一致を指定することができます。以下に、よく使用される特殊文字の説明を示します。
        
        - `.`: 任意の1文字に一致します。ただし、改行文字には一致しません。
        - `*`: 直前の文字が0回以上繰り返されることを指定します。
        - `+`: 直前の文字が1回以上繰り返されることを指定します。
        - `?`: 直前の文字が0回または1回出現することを指定します。
        - `^`: 行頭に一致します。
        - `$`: 行末に一致します。
        - `[]`: 括弧内のいずれかの文字に一致します。
        - `|`: 縦棒の前後のいずれかに一致します。
        - `()`: グループ化を行います。例えば、`(abc)+`は、`abc`が1回以上繰り返されることを指定します。
        
        正規表現で特殊文字を文字として検索するには、バックスラッシュ（`\\`）を使用してエスケープする必要があります。例えば、`.`を文字として検索する場合は、`\\.`と指定します。
