## わかったこと
- **プロセス管理**
    - **プロセスとは**
        
        Linuxにおいて、プロセスは実行中のプログラムを管理する単位です。
        
        シェルからコマンドを実行する場合のプロセスの親子関係と生成から消滅までの流れを説明し、プロセスの基本的な状態遷移を理解することは、システム管理やアプリケーション開発において重要です。
        
    - **スケジューリング**
        
        Linuxはマルチユーザ、マルチタスクのOSであり、プロセスの実行順序はスケジューラによって管理されています。各プロセスはRun Queue（待ち行列）で待機し、Nice値を使用して優先度を設定できます。
        
    - **フォアグランドジョブとバックグラウンドジョブ**
        
        Linuxでは、シェルが管理するプログラムの単位であるジョブをフォアグラウンドとバックグラウンドに切り替えることができ、処理に時間のかかるジョブを実行する場合やプログラムの編集作業を中断する場合に有効です。（以下例）
        
        - ジョブをバックグラウンドで実行することで、他の作業を同時に行える
        - プログラムの編集作業を中断して試行し、undoやredoを継続できる。
        
        Linuxでコマンドをバックグラウンドで起動するには、コマンドの末尾に&を追加するか、Ctrl + Zを押してからbgコマンドを使用します。
        
        - **コマンドのバックグラウンドでの起動方法**
            - コマンドの末尾に&を追加する方法
            
            ```bash
            command &
            ```
            
        - **実行中のコマンドをバックグラウンドに切り替える方法**
            
            Linuxで実行中のコマンドをバックグラウンドに切り替えるには、Ctrl + Zを押してからbgコマンドを使用します。
            
            以下は、この方法の例です。
            
            ```bash
            command
            Ctrl+Z
            bg
            
            ```
            
            これにより、現在実行中のコマンドがバックグラウンドで実行されるようになります。
            
    - **プロセス ID**
        
        Linuxのプロセスには、一意のプロセスID（PID）が割り当てられます。自分自身のPIDは$$を使用して取得できます。
        
        - **psコマンド**
            
            `ps`コマンドは、Linuxで実行中のプロセスの一覧を表示するために使用されるユーティリティです。以下は、`ps`コマンドの使用例です。
            
            - 実行中のプロセスを表示する
                
                ```bash
                ps aux
                ```
                
                このコマンドで表示される情報には、プロセスのID、親プロセスのID、CPU使用率、メモリ使用量、および実行中のプログラムの名前などが含まれます。詳細な情報を取得するには、`-ef`オプションを追加することができます。
                
                ```bash
                ps -ef
                ```
                
                このオプションを使用すると、ユーザー名、コマンドライン引数、およびプロセスの開始時刻などの詳細な情報が表示されます。
                
            - オプション:
                - **a**: このオプションは、すべてのユーザーのプロセスを表示します。
                - **e**: このオプションは、端末に関連付けられていないプロセスも含めて、すべてのプロセスを表示します。
                - **l**: このオプションは、UID、GID、PPID、CPU使用率、メモリ使用量など、各プロセスの詳細な情報を表示します。
                - **w**: このオプションは、各プロセスを起動するために使用された完全なコマンドラインを表示します。
            
            **$$とは**
            
            `$$`は、現在実行中のシェルのプロセスIDを表します。この変数は、スクリプト内で現在実行中のプロセスのIDを取得するために使用されます。
            
- **シグナル**
    
    Linuxには、プロセスを制御するためにシグナルというイベントをプロセスに送信する機能があり、代表的なシグナルにはシグナル番号およびシグナル名が割り当てられています。
    
    | シグナル番号 | シグナル名 | 意味 |
    | --- | --- | --- |
    | 1 | SIGHUP | ハングアップ |
    | 2 | SIGINT | 割り込み |
    | 3 | SIGQUIT | 終了 |
    | 9 | SIGKILL | 強制終了 |
    | 15 | SIGTERM | 終了 |
    
    ユーザーがプロセスにシグナルを送信する方法は3つあります。
    
    1. シェルに割り当てられたキーの入力（例：^C、^Z、^¥など）
    2. killコマンド（例：kill -HUP PID、kill -SIGINT PID、kill -9 PIDなど）
    3.  プログラムでkill()関数を呼び出す
    - **killコマンド**
        
        `kill`は、UnixまたはLinuxオペレーティングシステム上で実行中のプロセスにシグナルを送信するコマンドです。このコマンドは、プロセスを終了したり、プロセスに他のシグナルを送信したりするために使用されます。コマンドの構文は以下の通りです。
        
        ```
        kill [シグナルまたはオプション] [pid]
        
        ```
        
        `シグナル`は、プロセスに送信するシグナルです。シグナルが指定されていない場合、`TERM`シグナルが送信されます。`pid`は、シグナルを送信するプロセスのプロセスIDです。
        
        `kill`で送信できる一般的なシグナルには、以下のものがあります。
        
        - `TERM` (15): `kill`が送信するデフォルトのシグナルです。プロセスに正常に終了するように要求します。
        - `HUP` (1): このシグナルは、プロセスを再起動するために通常使用されます。
        - `INT` (2): ユーザーがCtrl+Cを押すと、割り込みシグナルが送信されます。
        - `KILL` (9): このシグナルは、プロセスを直ちに終了させます。
        
        `kill`コマンドで使用できるオプションには、以下があります。
        
        - `s signal`: プロセスに送信するシグナルを指定します。
        - `l`: 利用可能なすべてのシグナルを一覧表示します。
        - `p`: シグナルを受信するプロセスのプロセスIDを表示しますが、実際にシグナルを送信しません。
        - `a`: `init`プロセスを除くすべてのプロセスにシグナルを送信します。
    
- **top コマンドと pstree コマンド**
    
    ps コマンドに加えて、プロセスの状態を表示するための top コマンドがあります。
    
    top コマンドは、実行中のプロセスの状態をダイナミックに表示します。CPU やメモリの使用率などでソートしたり、指定したプロセスにシグナルを送信することもできます。
    
    また、プロセスの親子関係をツリー表示する pstree コマンドも使用できます。
    
    - **top コマンド**
        
        `top` コマンドは、Linux において実行中のプロセスの情報を表示するコマンドです。`top` コマンドで表示される情報には、プロセスの ID、プロセスの所有者、プロセスが使用している CPU の割合、プロセスが使用しているメモリの量などが含まれます。`top` コマンドは、プロセスの状態をダイナミックに表示するため、監視やトラブルシューティングに頻繁に使用されます。
        
        以下は、`top` コマンドの使用例です。
        
        ```bash
        top
        
        ```
        
        `top` コマンドで使用できるいくつかのオプションには、以下があります。
        
        - `d`: ディスプレイの更新間隔を設定します。
        - `p`: 特定の PID に関する情報のみを表示します。
        - `u`: 特定のユーザーが所有するプロセスに関する情報のみを表示します。
        - `S`: プロセスを CPU の使用率でソートします。
        - `M`: プロセスをメモリ使用量でソートします。
    - **pstree コマンド**
        
        `pstree` コマンドは、実行中のプロセスの親子関係をツリー状に表示するコマンドです。以下は、`pstree` コマンドの使用例です。
        
        ```bash
        pstree
        
        ```
        
        このコマンドで表示される情報には、プロセスの ID、プロセスの所有者、プロセスが使用している CPU の割合、プロセスが使用しているメモリの量、およびプロセスの親子関係などが含まれます。
